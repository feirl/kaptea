Background

A fleet and telematics business with a contact centre under pressure

Attrix manages telematics and fleet operations across a large customer base. Every day, drivers, fleet partners, and HR contacts ring in with operational queries, none of them simple, all of them time-sensitive.

The contact centre runs on Zendesk for ticketing and case management. The phone system was a separate stack. Agents took calls cold, then opened Zendesk to find or create the right ticket. Every call started with the agent asking for an account number, then for the nature of the query, before any actual help could begin.

The Challenge

The IVR and the CRM were not on speaking terms

Call volumes were rising. The team was good, but every interaction started from zero. The IVR captured menu selections, but those selections never made it into Zendesk. Agents took calls, asked the same opening questions, manually created tickets, and only then started solving the actual problem.

Attrix needed a way to enrich every Zendesk ticket with the caller's context the moment the call connected: who they are, what they're calling about, what menu options they chose. The agent should know all of that before they pick up.

The Solution

An IVR that briefs the agent before the agent picks up

Kaptea designed a custom IVR-to-Zendesk integration using Twilio Studio. The IVR captures the caller's identity, their reason for calling, and the menu options they selected, then automatically populates a Zendesk ticket with all of it before the call routes to an agent.

The agent answers with the full context already on screen. No data entry. No "Can I take your account number?" The conversation starts at the actual problem.

Twilio Studio IVR build

A custom IVR flow designed around Attrix's real call categories: drivers, fleet partners, and HR. Menu paths shaped by actual call patterns, not a generic template.

Zendesk auto-population

The IVR writes caller ID, call reason, and menu selections directly into the Zendesk ticket fields. Agents see everything they need the moment the call lands.

Reduced agent admin

Every interaction that previously started with two minutes of "Can I get your details?" now starts with the agent already briefed. The customer feels known. The agent feels prepared.

Scaled across the network

The integration handles the full 14,000-call monthly volume across 28 sales and service advisors, with no degradation as call patterns shift across the day.

Results

53 seconds back, on every call

Average handle time dropped by 53 seconds per call. With 14,000 monthly calls, that's roughly 200 hours of advisor time freed every month, time now spent solving the actual problem instead of opening tickets.

  • Average handle time down by 53 seconds

    Calls resolve faster because the agent already knows the caller, the reason, and the menu path. Less repetition, lower call duration, more capacity in the same headcount.

  • 14,000 monthly calls processed without manual ticket entry

    Every call lands in Zendesk pre-populated with caller and call-context fields. Agents stop doing data entry and start doing actual support.

  • 28 advisors supported, briefed on every call

    Sales and service advisors no longer take calls cold. Customer satisfaction lifts. Repeat callers get continuity of context across interactions.

  • Twilio + Zendesk delivered as one experience

    Two systems, one workflow. The agent never knows where the IVR ends and the CRM begins, and they never have to.

  • Customer satisfaction up, agent workload down

    The same advisors handling the same calls, but with the friction taken out of the first 60 seconds. The metric Attrix optimised for; the result they got.
